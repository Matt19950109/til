## コンフリクトの解決方法
- ファイルで自分が修正したい箇所を反映させる
- git add
- git commit

## コンフリクト関連の事故が起きにくい運用ルール

- 複数人で同じファイルを変更しない
- pullやmergeをする前に変更中の状態をなくしておく
- pullするときは、pullするブランチに移動してから実施する
- コンフリクトしても慌てない

## ブランチを変更・削除する

- ブランチの変更
  　git branch -m <ブランチ名>
  
- ブランチの削除
    ①安全な削除
    git branch -d <ブランチ名>
  
    ②強制削除
    git branch -D <ブランチ名>
